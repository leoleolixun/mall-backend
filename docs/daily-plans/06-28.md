这个文件放通用分页响应。它不是商品专属结构，后面订单列表、用户列表、地址列表也可以复用：

package dto

	List     []T   `json:"list"`
	Page     int   `json:"page"`

```go

```

如果你暂时不想用泛型，也可以先写死商品专用响应：

```go
type ProductPageResponse struct {

```

## 第 13 步：商品详情 Redis 缓存

商品详情缓存可以放在 `ProductService.Detail` 中。

缓存 key：

```go
key := fmt.Sprintf("mall:product:detail:%d", id)
```

读取流程：

```text
1. Redis GET
2. 命中则 json.Unmarshal 后返回
3. 未命中则查 MySQL
4. 查到后 json.Marshal 写入 Redis，TTL 5 分钟
5. 查不到则写入短 TTL 空值，比如 1 分钟
```

你需要给 `ProductService` 注入 Redis：

```go
type productService struct {
	productRepo repository.ProductRepository
	redis       *redis.Client
}
```

缓存 TTL：

```go
const productDetailCacheTTL = 5 * time.Minute
const emptyCacheTTL = time.Minute
```

今天不需要做复杂缓存一致性，因为还没有后台商品修改接口。

func NewRouter(
	healthHandler *handler.HealthHandler,
	categoryHandler *handler.CategoryHandler,
	productHandler *handler.ProductHandler,

	r := gin.New()

	r.Use(gin.Logger())

	{
		api.GET("/categories", categoryHandler.List)

## 常见问题

### 1. AutoMigrate 没建表

检查：

- `main.go` 是否调用了 `bootstrap.AutoMigrate(db)`
- model 是否传了指针：`&model.Product{}`
- MySQL 用户是否有建表权限
- 当前连接的是不是你以为的数据库

### 2. 分类或商品重复插入

说明 seed 没做幂等。

先简单处理：

- 如果默认商户存在，则跳过全部 seed
- 或者按名称查询，不存在再插入

### 3. 商品详情一直查不到

检查：

- 商品 ID 是否存在
- `merchant_id` 是否为 1
- 商品状态是否为上架
- SKU 状态是否启用
- 是否被软删除

### 4. Redis 没有缓存 key

检查：

- 商品详情接口是否真的被调用
- `ProductService.Detail` 是否写入 Redis
- Redis DB 是否和 `config.yaml` 一致
- key 前缀是否写成 `mall:product:detail:`

### 5. import cycle not allowed

说明包之间循环依赖。

今天保持方向：

```text
handler -> service -> repository -> model
```

不要让 `model` import `repository`，也不要让 `repository` import `service`。
