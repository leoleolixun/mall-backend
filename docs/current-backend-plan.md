# go-mall 当前后端开发计划

## 当前定位

`docs/openapi.yaml` 是完整接口蓝图，包含 MVP、后续增强、多商户、商家后台和平台后台。

当前后端实现不要按完整接口蓝图铺开。买家侧单商户 MVP 已基本完成，下一步进入“真实联调 + 商家端最小闭环”：

```text
认证 -> 商品 -> 地址 -> 购物车 -> 订单 -> 支付单 -> 支付宝网页支付 -> 商家发货
```

## 当前必须完成

### 1. 稳定已实现模块

- 认证注册和微信首次登录必须使用事务
- 地址默认地址更新必须使用事务
- 远程 MySQL 默认不自动 `AutoMigrate`
- 远程 MySQL 默认不自动写种子数据
- 地址和购物车接口必须注册路由并能通过 Swagger 查看

### 2. 订单模块

已实现这些接口：

```http
POST /api/v1/orders/preview
POST /api/v1/orders
GET  /api/v1/orders
GET  /api/v1/orders/{id}
POST /api/v1/orders/{id}/cancel
POST /api/v1/orders/{id}/pay
```

订单模块必须保证：

- [x] 创建订单使用 MySQL 事务
- [x] 订单主表、订单明细、库存扣减、购物车清理保持一致
- [x] 订单保存商品快照、SKU 快照、地址快照
- [x] 库存扣减使用条件更新
- [x] 用户只能访问自己的订单
- [x] 订单状态流转明确

### 3. 最小测试

优先补这些测试或可重复脚本：

- JWT 中间件
- 账号密码登录
- 创建订单成功
- 库存不足不能下单
- 重复提交不能重复建单
- 已支付订单不能取消
- 已取消订单不能支付

### 4. 支付模块

已实现这些接口：

```http
POST /api/v1/payments
GET  /api/v1/payments/{payment_no}
POST /api/v1/payments/{payment_no}/mock-complete
POST /api/v1/payments/alipay/notify
POST /api/v1/orders/{id}/pay
```

支付模块当前保证：

- [x] 待支付订单可以创建支付单
- [x] 同一订单同一渠道重复创建支付单会返回已有待支付支付单
- [x] 订单 mock 支付会创建或复用 mock 支付单，并更新订单为已支付
- [x] 支付单 mock 完成支付会更新支付单和订单状态
- [x] mock/wechat 渠道会返回不同的 mock 支付参数
- [x] 支付宝 PC 网页支付会返回真实 `pay_url`
- [x] 支付宝异步通知会验签、校验金额，并更新支付单和订单状态
- [x] 已支付订单不能再创建支付单
- [x] 用户只能查询自己的支付单

## 暂时不要实现

这些已经在 OpenAPI 里作为规划接口存在，但当前不要实现：

- 收藏
- 优惠券
- 真实微信支付
- 售后退款
- 平台后台
- 多商户拆单

这些可以作为下一阶段逐步实现：

- 商家端订单列表和订单详情
- 商家端发货接口
- 用户端物流信息查询
- 支付宝支付主动查询接口
- 订单超时取消任务

## 配置要求

连接远程 MySQL 时建议：

```yaml
app:
  auto_migrate: false
  seed_data: false
```

本地开发库需要自动建表和种子数据时再改为：

```yaml
app:
  auto_migrate: true
  seed_data: true
```

真实配置文件 `config.yaml` 不提交到 Git。
