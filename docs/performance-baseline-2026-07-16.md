# go-mall 性能基线（2026-07-16）

## 测试范围

- 目标：`https://mall.leedu.ac.cn/api/v1`
- 基线提交：`918c302` 加当前 M6 未提交改动
- 数据量：公开商品 3 个，详情商品 ID `3`
- 客户端：macOS Darwin 25.5.0 arm64，Apple M4，16 GiB 内存
- 路径：客户端公网网络 -> Nginx TLS -> Go API -> 线上 MySQL/Redis
- 服务端 CPU/内存：本机没有服务器 SSH 私钥，未能只读采集；V1 发布验收必须补录
- 写请求：未运行，避免在生产数据上创建订单或修改库存

## 参数

```bash
BASE_URL=https://mall.leedu.ac.cn/api/v1 \
TEST_PRODUCT_ID=3 \
READ_REQUESTS=200 \
READ_CONCURRENCY=50 \
READ_WARMUP_REQUESTS=50 \
./scripts/benchmark-api.sh
```

预热请求不计入延迟统计，只用于建立与并发数一致的 TLS/HTTP 连接。订单写压测不会自动预热，避免重复使用幂等令牌。

## 结果

| 接口 | 请求数 | 并发 | RPS | P50 | P95 | P99 | HTTP 200 | 错误 |
| --- | ---: | ---: | ---: | ---: | ---: | ---: | ---: | ---: |
| `GET /products?page=1&page_size=20` | 200 | 50 | 196.39 | 191ms | 323ms | 543ms | 200 | 0 |
| `GET /products/3` | 200 | 50 | 682.10 | 63ms | 84ms | 102ms | 200 | 0 |

两条商品读接口的 P95 均低于 500ms，50 并发下没有 5xx 或客户端错误。

## 首次建连对照

未预热时同日结果：列表 P95 `428ms`，详情 P95 `534ms`。详情超过阈值主要发生在 50 条首次 TLS 连接同时建立阶段，因此基准工具增加了独立预热阶段。公网首屏仍会承受 TLS 建连开销，不能用预热结果代替真实用户体验监控。
