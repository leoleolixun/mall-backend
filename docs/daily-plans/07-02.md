## 详细执行步骤

### 第 1 步：实现订单查询

接口：

```http
GET /api/v1/orders
GET /api/v1/orders/:id
```

要求：

```text
只能查询当前登录用户自己的订单
订单详情包含 order_items
列表支持分页
```

### 第 2 步：实现取消订单

接口：

```http
POST /api/v1/orders/:id/cancel
```

流程：

```text
查询订单并校验 user_id
校验状态必须是待支付
事务内更新订单为已取消
事务内恢复 SKU 库存
写入 cancelled_at
```

### 第 3 步：实现模拟支付

接口：

```http
POST /api/v1/orders/:id/pay
```

流程：

```text
查询订单并校验 user_id
校验状态必须是待支付
更新订单为已支付
写入 paid_at
```

### 订单查询

实现：

```http
GET /api/v1/orders
GET /api/v1/orders/:id
```

要求：

- 只能查询当前用户订单
- 订单详情包含订单明细
- 列表支持分页

### 取消订单

实现：

```http
POST /api/v1/orders/:id/cancel
```

要求：

- 只有待支付订单可以取消
- 取消订单恢复库存
- 状态更新和库存恢复放在同一个事务中

### 模拟支付

实现：

```http
POST /api/v1/orders/:id/pay
```

要求：

- 只有待支付订单可以支付
- 支付后状态变为已支付
- 写入 `paid_at`
