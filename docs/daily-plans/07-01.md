### 第 1 步：定义订单模型


### 第 2 步：定义订单状态

不要用多个布尔字段，使用明确状态：

```text
1 待支付
2 已支付
3 已发货
4 已完成
5 已取消
```

第一版只用：

```text
待支付
已支付
已取消
```

### 第 3 步：实现订单预览

接口：

```http
POST /api/v1/orders/preview
```

流程：

```text
校验地址属于当前用户
读取购物车或请求商品
查询 SKU 和商品
校验商品上架、SKU 启用、库存充足
计算金额
生成幂等 token
写入 Redis
返回预览结果
```

### 第 4 步：实现创建订单事务

接口：

```http
POST /api/v1/orders
```

事务内完成：

```text
校验幂等 token
校验地址
查询商品和 SKU
条件扣减库存
创建订单主表
创建订单明细
保存地址快照
保存商品快照
清理购物车
标记幂等 token
```

库存扣减建议使用条件更新：

```sql
UPDATE product_skus
SET stock = stock - ?
WHERE id = ? AND stock >= ?
```

如果影响行数为 0，说明库存不足。

### 第 5 步：重点自测

必须测：

```text
正常创建订单
库存不足
地址不属于当前用户
重复提交
创建订单后库存减少
创建订单后购物车清理
```

### 订单预览

实现：

```http
POST /api/v1/orders/preview
```

预览逻辑：

- 校验地址属于当前用户
- 校验购物车或请求中的 SKU
- 查询商品和 SKU 当前状态
- 校验库存
- 计算总金额
- 返回商品明细
- 返回地址信息
- 返回幂等 token
- 幂等 token 写入 Redis

### 创建订单

实现：

```http
POST /api/v1/orders
```

创建逻辑必须在事务中完成：

- 校验幂等 token
- 校验地址
- 查询 SKU
- 校验商品状态
- 校验库存
- 创建订单主表
- 创建订单明细
- 保存商品快照
- 保存地址快照
- 扣减库存
- 清理购物车中已下单商品
- 将幂等 token 标记为已使用或绑定订单号
