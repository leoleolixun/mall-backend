# 2026-06-29：第 3 天 - 用户体系、账号密码登录、微信模拟登录

## 今日目标

完成统一用户体系，支持账号密码登录和微信模拟登录。两种登录方式最终都映射到同一个 `users.id`。

## 今日学习

### Go 认证相关

- interface 的基本设计方式
- bcrypt 密码哈希
- JWT access token
- refresh token 的基本思路
- Gin middleware
- 从 Gin Context 中读取当前用户

### 用户体系设计

- `users` 保存用户主体
- `user_auths` 保存登录方式
- provider 设计
- 微信 openid 与用户绑定关系

### Redis

- refresh token 存储
- 登录失败计数
- TTL

## 今日完成

### 数据表

定义并迁移：

```text
users
user_auths
```

`user_auths` 至少包含：

```text
user_id
provider
provider_uid
credential
```

唯一约束：

```text
provider + provider_uid
```

### 认证接口

实现：

```http
POST /api/v1/auth/register
POST /api/v1/auth/login/password
POST /api/v1/auth/login/wechat/mock
POST /api/v1/auth/refresh
POST /api/v1/auth/logout
GET  /api/v1/me
```

### JWT 中间件

- 校验 Authorization header
- 解析 user_id
- 写入 Gin Context
- 未登录返回统一错误

### Redis

- 登录成功后写入 refresh token
- 登录失败时增加失败计数
- 失败次数过多时短时间拒绝登录
