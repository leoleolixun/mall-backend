## 详细执行步骤

### 第 1 步：定义用户模型

创建：

```text
internal/model/user.go
internal/model/user_auth.go
```

`users` 保存用户主体信息，`user_auths` 保存登录方式。

今天不要拆成 `password_users`、`wechat_users` 两套用户系统。正确方向是：

```text
users
user_auths
```

两种登录方式最终都映射到同一个 `users.id`。

### 第 2 步：补 AutoMigrate

把新模型加入：

```text
internal/bootstrap/migrate.go
```

新增：

```go
&model.User{},
&model.UserAuth{},
```

### 第 3 步：实现密码工具

创建：

```text
pkg/password/password.go
```

```go
func HashPassword(raw string) (string, error)
func CheckPassword(hash string, raw string) bool
```

```text
golang.org/x/crypto/bcrypt
```

### 第 4 步：实现 JWT 工具

创建：

```text
pkg/jwt/jwt.go
```

先支持：

```go
func GenerateAccessToken(userID int64) (string, error)

```

第一版可以先把 secret 写进配置，后面再支持环境变量。

### 第 5 步：实现 AuthRepository

创建：

```text
internal/repository/auth_repository.go
```

至少实现：

```go
CreateUser(ctx, user)
CreateUserAuth(ctx, auth)
FindAuthByProvider(ctx, provider, providerUID)
FindUserByID(ctx, userID)
```

### 第 6 步：实现 AuthService

创建：

```text
internal/service/auth_service.go
```

实现：

```text
注册
账号密码登录
微信模拟登录
刷新 token
退出登录
获取当前用户
```

账号密码注册流程：

```text
校验 username/password
查询 username 是否已存在
bcrypt hash 密码
创建 users
创建 user_auths(provider=password)
```

微信模拟登录流程：

```text
接收 mock_openid
查询 user_auths(provider=wechat_miniprogram, provider_uid=mock_openid)
存在则返回对应用户
不存在则创建 users + user_auths
签发 JWT
```

### 第 7 步：实现 JWT 中间件

创建：

```text
internal/middleware/auth.go
```

中间件负责：

```text
读取 Authorization: Bearer <token>
解析 token
把 user_id 写入 gin.Context
解析失败时返回未登录
```

### 第 8 步：实现 AuthHandler

创建：

```text
internal/handler/auth_handler.go
```

实现接口：

```http
POST /api/v1/auth/register
POST /api/v1/auth/login/password
POST /api/v1/auth/login/wechat/mock
POST /api/v1/auth/refresh
POST /api/v1/auth/logout
GET  /api/v1/me
```

### 第 9 步：接入 Redis

Redis key 设计：

```text
mall:auth:refresh:{user_id}:{token_id}

```

今天至少完成：

```text
refresh token 写入 Redis
```

### JWT 中间件

- 校验 Authorization header
- 解析 user_id
- 写入 Gin Context

- 登录失败时增加失败计数
- 失败次数过多时短时间拒绝登录
