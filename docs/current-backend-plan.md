# go-mall 当前后端开发计划

## 当前定位

`docs/openapi.yaml` 是完整接口蓝图，包含 MVP、后续增强、多商户、商家后台和平台后台。

买家侧单商户 V1、商家端基础闭环、M6 工程门禁和 M7 移动端工程已经完成。M8-D 交易级支付、不可变支付分配、退款冲正、佣金快照、结算账本及 PC/管理端/UniApp 适配已完成工程验收；旧单商户订单入口继续作为兼容接口保留，并在内部创建单订单交易：

```text
公开商户 -> 跨商户商品 -> 跨商户购物车
  -> 版本化迁移 -> 交易单 -> 商户子订单 -> 支付分配 -> 结算台账
```

## 当前必须完成

### 2. 订单模块

已实现这些接口：

```http
POST /api/v1/orders/preview
POST /api/v1/orders
GET  /api/v1/orders
GET  /api/v1/orders/{id}
POST /api/v1/orders/{id}/cancel
POST /api/v1/orders/{id}/confirm
GET  /api/v1/orders/{id}/logistics
```

`POST /api/v1/orders/{id}/pay` 是旧版模拟支付入口，仅在本地显式开启 `payment.mock_enabled` 时注册，生产环境不开放。

多商户结算使用新的交易接口：

```http
POST /api/v1/trades/preview
POST /api/v1/trades
GET  /api/v1/trades
GET  /api/v1/trades/{id}
POST /api/v1/trades/{id}/cancel
```

一次交易按商户原子创建多张子订单；任一商品、库存、商户券或报价校验失败时整笔回滚。交易支付统一使用 `trade_id` 创建一张聚合支付单，并按每张子订单生成不可变支付分配；不能拿第一张子订单调用旧支付接口冒充合并支付。

订单模块必须保证：

- [x] 创建订单使用 MySQL 事务
- [x] 订单主表、订单明细、库存扣减、购物车清理保持一致
- [x] 订单保存商品快照、SKU 快照、地址快照
- [x] 库存扣减使用条件更新
- [x] 用户只能访问自己的订单
- [x] 订单状态流转明确
- [x] 买家可以查询基础物流信息并主动确认收货
- [x] 已发货订单支持定时自动完成
- [x] 当前不接入第三方实时快递轨迹

### 4. 支付模块

已实现这些接口：

```http
POST /api/v1/payments
GET  /api/v1/payments/{payment_no}
POST /api/v1/payments/alipay/notify
POST /api/v1/payments/{payment_no}/sync
```

模拟支付接口仅在本地显式开启 `payment.mock_enabled` 时注册；生产环境默认关闭。

支付模块当前保证：

- [x] 待支付交易可以创建一张聚合支付单并生成子订单支付分配
- [x] 同一交易只保留一个待支付意图；相同渠道和场景重复请求返回原支付单，切换支付方式会被拒绝
- [x] 支付成功原子更新交易、全部子订单和支付单，回调金额与支付分配总额一致
- [x] 子订单退款只能冲减自身支付分配，不影响同一交易内其他商户
- [x] 旧 `order_id` 支付仅兼容单订单交易，多商户子订单必须使用 `trade_id`
- [x] 模拟支付由配置开关控制，路由和 Service 双层限制
- [x] 未实现的微信支付不会生成伪支付参数或伪二维码
- [x] 支付宝 PC 网页支付会返回真实 `pay_url`
- [x] 支付宝异步通知会验签、校验金额，并更新支付单和订单状态
- [x] 已支付订单不能再创建支付单
- [x] 用户只能查询自己的支付单

### 5. 买家个人中心

已实现这些接口：

```http
GET    /api/v1/coupons
POST   /api/v1/coupons/{id}/claim
GET    /api/v1/me/coupons
GET    /api/v1/favorites/products
POST   /api/v1/favorites/products
DELETE /api/v1/favorites/products/{product_id}
GET    /api/v1/after-sales
POST   /api/v1/after-sales/{id}/cancel
```

收藏接口使用用户、商户和商品联合唯一索引，新增时保存商品真实商户，重复收藏和重复取消均按幂等成功处理；收藏列表跨商户返回仍在售且商户启用的商品。

## 配置要求

连接远程 MySQL 时建议：

```yaml
app:
  auto_migrate: false
  seed_data: false
```

本地开发库需要自动建表和种子数据时再改为：

```yaml
app:
  auto_migrate: true
  seed_data: true
```

真实配置文件 `config.yaml` 不提交到 Git。
