# 2026-06-30：第 4 天 - 收货地址和 Redis 购物车

## 今日目标

完成登录后的用户基础能力：收货地址和 Redis 购物车。

购物车从第一版开始使用 Redis，但创建订单时仍然必须以 MySQL 中的商品、SKU、库存、价格为准。

## 今日学习

### Go Web

- 从中间件中获取当前用户
- 请求参数校验
- 用户数据隔离
- Handler 中只处理入参和出参

### Redis

- Hash 数据结构
- `HSET`
- `HINCRBY`
- `HGETALL`
- `HDEL`
- Redis 数据与 MySQL 数据组合返回

### 业务设计

- 地址属于用户
- 购物车只是临时状态
- 商品下架后的购物车展示策略
- SKU 库存校验

## 今日完成

### 地址表

定义并迁移：

```text
addresses
```

### 地址接口

实现：

```http
GET    /api/v1/addresses
POST   /api/v1/addresses
PUT    /api/v1/addresses/:id
DELETE /api/v1/addresses/:id
PUT    /api/v1/addresses/:id/default
```

### 购物车 Redis 结构

使用：

```text
key: mall:cart:{user_id}
type: hash
field: sku_id
value: quantity
```

### 购物车接口

实现：

```http
GET    /api/v1/cart
POST   /api/v1/cart/items
PUT    /api/v1/cart/items/:sku_id
DELETE /api/v1/cart/items/:sku_id
DELETE /api/v1/cart
```

购物车列表返回：

- 商品 ID
- SKU ID
- 商品名称
- SKU 名称
- SKU 图片
- 当前价格
- 数量
- 小计
- 商品状态
- 库存状态
