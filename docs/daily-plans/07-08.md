# 2026-07-08：创建订单事务、订单查询、取消订单、模拟支付

```text

今天重点不是接口数量，而是事务、库存和状态流转正确。

	Pay(ctx context.Context, userID int64, id int64) (*dto.OrderResponse, error)
}
```

## 第 2 步：实现创建订单事务

接口：

```http
POST /api/v1/orders
```

事务内必须完成：

```text
1. 校验幂等 token
2. 校验地址属于当前用户
3. 查询 SKU 和商品
4. 校验商品状态、SKU 状态、库存
5. 条件扣减库存
6. 创建订单主表
7. 创建订单明细
8. 清理购物车中已下单 SKU
9. 标记幂等 token 已使用或删除 token
```

库存扣减必须使用条件更新：

```sql
UPDATE product_skus
SET stock = stock - ?
WHERE id = ? AND stock >= ?
```

判断：

```text
RowsAffected == 0 表示库存不足
```

## 第 3 步：实现订单查询

接口：

```http
GET /api/v1/orders
GET /api/v1/orders/{id}
```

要求：

- 只能查询当前用户自己的订单
- 列表支持分页
- 详情包含订单明细
- 不允许通过订单 ID 看到别人的订单

查询条件必须包含：

```text
user_id = 当前用户 ID
```

## 第 4 步：实现取消订单

接口：

```http
POST /api/v1/orders/{id}/cancel
```

规则：

```text
只有待支付订单可以取消
已支付订单不能取消
已取消订单不能重复取消
```

事务内完成：

```text
1. 查询订单并锁定或确保状态条件
2. 校验 user_id
3. 更新订单状态为已取消
4. 写入 cancelled_at
5. 恢复 SKU 库存
```

## 第 5 步：实现模拟支付

接口：

```http
POST /api/v1/orders/{id}/pay
```

规则：

```text
只有待支付订单可以支付
已取消订单不能支付
已支付订单不能重复支付
```

第一版可以只更新：

```text
status = 已支付
paid_at = 当前时间
```

## 第 6 步：实现 OrderHandler

在：

```text
internal/handler/order_handler.go
```

补齐：

```go
func (h *OrderHandler) Create(c *gin.Context)
func (h *OrderHandler) List(c *gin.Context)
func (h *OrderHandler) Detail(c *gin.Context)
func (h *OrderHandler) Cancel(c *gin.Context)
func (h *OrderHandler) Pay(c *gin.Context)
```

所有接口都必须：

- 从 JWT 获取当前用户
- 使用 `c.Request.Context()`
- 不在 Handler 写业务逻辑
