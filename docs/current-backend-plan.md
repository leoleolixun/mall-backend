# go-mall 当前后端开发计划

## 当前定位

`docs/openapi.yaml` 是完整接口蓝图，包含 MVP、后续增强、多商户、商家后台和平台后台。

当前后端实现不要按 67 个接口铺开，先完成买家侧单商户 MVP：

```text
认证 -> 商品 -> 地址 -> 购物车 -> 订单 -> 模拟支付
```

## 当前必须完成

### 1. 稳定已实现模块

- 认证注册和微信首次登录必须使用事务
- 地址默认地址更新必须使用事务
- 远程 MySQL 默认不自动 `AutoMigrate`
- 远程 MySQL 默认不自动写种子数据
- 地址和购物车接口必须注册路由并能通过 Swagger 查看

### 2. 订单模块

优先实现这些接口：

```http
POST /api/v1/orders/preview
POST /api/v1/orders
GET  /api/v1/orders
GET  /api/v1/orders/{id}
POST /api/v1/orders/{id}/cancel
POST /api/v1/orders/{id}/pay
```

订单模块必须保证：

- 创建订单使用 MySQL 事务
- 订单主表、订单明细、库存扣减、购物车清理保持一致
- 订单保存商品快照、SKU 快照、地址快照
- 库存扣减使用条件更新
- 用户只能访问自己的订单
- 订单状态流转明确

### 3. 最小测试

优先补这些测试或可重复脚本：

- JWT 中间件
- 账号密码登录
- 创建订单成功
- 库存不足不能下单
- 重复提交不能重复建单
- 已支付订单不能取消
- 已取消订单不能支付

## 暂时不要实现

这些已经在 OpenAPI 里作为规划接口存在，但当前不要实现：

- 收藏
- 优惠券
- 真实微信支付
- 支付回调
- 物流轨迹
- 售后退款
- 商家后台
- 平台后台
- 多商户拆单

## 当前开发顺序

1. 修复已实现模块的数据一致性问题
2. 自测地址和购物车接口
3. 实现订单模型和 DTO
4. 实现订单 Repository
5. 实现订单预览
6. 实现创建订单事务
7. 实现订单查询、取消、模拟支付
8. 补最小测试和验收脚本
9. 再考虑优惠券、支付、售后等后续模块

## 配置要求

连接远程 MySQL 时建议：

```yaml
app:
  auto_migrate: false
  seed_data: false
```

本地开发库需要自动建表和种子数据时再改为：

```yaml
app:
  auto_migrate: true
  seed_data: true
```

真实配置文件 `config.yaml` 不提交到 Git。
